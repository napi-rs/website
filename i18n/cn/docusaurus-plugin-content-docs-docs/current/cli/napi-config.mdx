The config schema of **NAPI-RS**.

:::info

All the fields in `napi` is optional.

:::

## Schema

```ts
{
  napi?: {
    name?: string
    triples?: {
      default?: boolean,
      additional?: string[]
    },
    package?: {
      name?: string
    }
  }
}
```

| Filed                | Description                                                                                                                                                                                                               |
| -------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `name`               | The binary file name in generated `.node` file. Eg <Green>`[NAME].[TRIPLE?].node`</Green>                                                                                                                                 |
| `triples.default`    | Whether to enable the default triples. <br />The default triples is `['x86_64-apple-darwin', 'x86_64-unknown-linux-gnu', 'x86_64-pc-windows-msvc']`.<br />The default value of `triples.default` is <Green>`true`</Green> |
| `triples.additional` | Additional triples besides the default triples you want to build. Target triples could be found in the output of `rustup target list` command.                                                                            |
| `package.name`       | Override the `name` field in `package.json`. See [Build#js-package-name](./build#note-for---js-package-name) for usage.                                                                                                   |

## What is `target tripe`

See [rustc/platform-support](https://doc.rust-lang.org/nightly/rustc/platform-support.html) and [LLVM/CrossCompilation](https://clang.llvm.org/docs/CrossCompilation.html#target-triple)

> Targets are identified by their "target triple" which is the string to inform the compiler what kind of output should be produced.

> The triple has the general format `<arch><sub>-<vendor>-<sys>-<abi>`, where:
>
> - `arch` = `x86_64`, `i386`, `arm`, `thumb`, `mips`, etc.
> - `sub` = for ex. on ARM: `v5`, `v6m`, `v7a`, `v7m`, etc.
> - `vendor` = `pc`, `apple`, `nvidia`, `ibm`, etc.
> - `sys` = `none`, `linux`, `win32`, `darwin`, `cuda`, etc.
> - `abi` = `eabi`, `gnu`, `android`, `macho`, `elf`, etc.
