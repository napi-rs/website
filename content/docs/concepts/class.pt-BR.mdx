## Escolha a forma JavaScript correta

`#[napi]` em uma struct cria uma classe JavaScript com identidade nativa e mĂŠtodos. Outros atributos de struct criam formas de valor:

| DeclaraĂ§ĂŁo Rust | RepresentaĂ§ĂŁo JavaScript | Use para |
| --- | --- | --- |
| `#[napi] struct` | InstĂ˘ncia de classe apoiada por um valor Rust | Objetos nativos com estado, mĂŠtodos, identidade e referĂŞncias |
| `#[napi(object)] struct` | Objeto simples copiado de/para uma struct Rust prĂłpria | Registros, opĂ§Ăľes e formas de configuraĂ§ĂŁo |
| `#[napi(transparent)] struct Wrapper(T)` | O valor interno `T` | Newtypes Rust que nĂŁo devem adicionar um wrapper JavaScript |
| Struct de tupla `#[napi(array)]` | Array JavaScript / tupla TypeScript | Dados posicionais fixos |

Consulte [ConversĂľes de tipos](/pt-BR/docs/concepts/type-conversions) para as regras de direĂ§ĂŁo e propriedade e [Atributos `#[napi]`](/pt-BR/docs/concepts/napi-attributes) para todos os controles de forma.

## `Constructor`

### `constructor` padrĂŁo

Se todos os campos em uma struct `Rust` forem `pub`, entĂŁo vocĂŞ pode usar `#[napi(constructor)]` para fazer com que a `struct` tenha um `constructor` padrĂŁo.

```rust filename="lib.rs"

  pub name: String,
  pub kind: u32,
}
```

```ts filename="index.d.ts"

  name: string
  kind: number

}
```

Todo campo pĂşblico faz parte da API JavaScript: o napi-rs gera um getter e, a menos que o campo tenha `#[napi(readonly)]`, um setter. Portanto, seu tipo Rust deve aceitar a direĂ§ĂŁo de conversĂŁo JavaScript gerada. Mantenha o estado somente nativo privado, como `count` no exemplo de construtor personalizado abaixo.

### `constructor` personalizado

Se vocĂŞ quiser definir um `constructor` personalizado, pode usar `#[napi(constructor)]` na sua constructor `fn` no bloco `impl` da struct.

```rust filename="lib.rs"

pub struct JsQueryEngine {
  count: u32,
}

#[napi]
impl JsQueryEngine {
  #[napi(constructor)]
  pub fn new() -> Self {
    JsQueryEngine { count: 0 }
  }
}
```

```ts filename="index.d.ts"

  constructor()
}
```

<Callout type="warning" emoji="â ď¸">
  **NAPI-RS** atualmente nĂŁo suporta `private constructor`. Seu construtor
  personalizado deve ser `pub` em Rust.
</Callout>

## Factory

AlĂŠm do `constructor`, vocĂŞ tambĂŠm pode definir mĂŠtodos de fĂĄbrica na `Class` usando `#[napi(factory)]`.

  uma instĂ˘ncia (`new`) da `Class` em JavaScript, um erro serĂĄ lanĂ§ado.


## `class method`

VocĂŞ pode definir um mĂŠtodo de classe JavaScript com `#[napi]` em um mĂŠtodo de struct em **Rust**.

```rust filename="lib.rs"

  count: u32,
}

  pub fn with_initial_count(count: u32) -> Self {
    JsQueryEngine { count }
  }

  #[napi]

    Ok(format!("{query}: {}", self.count))
  }

  #[napi]
  pub fn status(&self) -> napi::Result<u32> {
    Ok(self.count)
  }
}
```

```ts filename="index.d.ts"
export class QueryEngine {
  static withInitialCount(count: number): QueryEngine
  constructor()
  query(query: string): Promise<string>
  status(): number
}
```

<Callout type="warning" emoji="â ď¸">
  `async fn` precisa dos recursos `napi4` e `tokio_rt` habilitados.
</Callout>

<Callout>
Qualquer  `fn` em `Rust` que retorne `Result<T>` serĂĄ tratado como `T` em JavaScript/TypeScript. Se o `Result<T>` for `Err`, um erro JavaScript serĂĄ lanĂ§ado.
</Callout>

## `Getter`

Defina [um `getter` de classe JavaScript](https://developer.mozilla.org/en-US/docs/Web/JavaScript/Reference/Functions/get) usando `#[napi(getter)]`. A `fn` Rust deve ser um mĂŠtodo de struct, nĂŁo uma funĂ§ĂŁo associada.

```rust filename="lib.rs"

  count: u32,
}

#[napi]

  #[napi(factory)]
  pub fn with_initial_count(count: u32) -> Self {
    JsQueryEngine { count }
  }

  /// MĂŠtodo de classe
  #[napi]
  pub async fn query(&self, query: String) -> napi::Result<String> {
    Ok(format!("{query}: {}", self.count))
  }

  #[napi(getter)]
  pub fn status(&self) -> napi::Result<u32> {
    Ok(self.count)
  }
}
```

```ts {4} filename="index.d.ts"
export class QueryEngine {
  static withInitialCount(count: number): QueryEngine
  constructor()
  get status(): number
}
```

## `Setter`

Defina [`setter` de classe JavaScript](https://developer.mozilla.org/en-US/docs/Web/JavaScript/Reference/Functions/set) usando `#[napi(setter)]`. A `fn` Rust deve ser um mĂŠtodo de struct, nĂŁo uma funĂ§ĂŁo associada.

```rust filename="lib.rs"
    Ok(format!("{query}: {}", self.count))

  pub fn status(&self) -> napi::Result<u32> {
    Ok(self.count)
  }
```

## Class como argumento

`Class` ĂŠ diferente de [`Object`](./object). O valor Rust ĂŠ encapsulado por uma instĂ˘ncia JavaScript e gerenciado pelo coletor de lixo desse ambiente. Somente campos pĂşblicos da struct se tornam propriedades JavaScript. Eles sĂŁo gravĂĄveis por padrĂŁo porque o napi-rs gera os dois acessores; `#[napi(readonly)]` elimina o setter, e `#[napi(skip)]` elimina ambos. Campos privados continuam sendo detalhes da implementaĂ§ĂŁo nativa. Um campo gravĂĄvel precisa de `ToNapiValue` e `FromNapiValue`; um campo readonly precisa apenas de `ToNapiValue`. Consulte a [referĂŞncia de atributos de campo](/pt-BR/docs/concepts/napi-attributes#fields), inclusive a limitaĂ§ĂŁo da forma abreviada `#[napi(constructor)]`.

Para instĂ˘ncias de classe aninhadas, arrays de instĂ˘ncias de classe e `ClassInstance<T>`, consulte a [seĂ§ĂŁo de classes da referĂŞncia de conversĂŁo](/pt-BR/docs/concepts/type-conversions#objects-classes-and-custom-shapes).

## Atributos de propriedade

Os atributos padrĂŁo da propriedade sĂŁo `writable = true`, `enumerable = true` e `configurable = true`. VocĂŞ pode controlar os atributos da propriedade atravĂŠs do macro `#[napi]`:

## LĂłgica de FinalizaĂ§ĂŁo personalizada

O NAPI-RS descartarĂĄ a struct Rust envolvida no objeto JavaScript quando o objeto JavaScript for "garbage collected". VocĂŞ tambĂŠm pode especificar uma lĂłgica de finalizaĂ§ĂŁo personalizada para a struct Rust.

```rust filename="lib.rs"
use napi::bindgen_prelude::*;
use napi_derive::napi;

#[napi(custom_finalize)]
pub struct CustomFinalize {
  width: u32,
  height: u32,
  inner: Vec<u8>,
}

#[napi]
impl CustomFinalize {
  #[napi(constructor)]
  pub fn new(mut env: Env, width: u32, height: u32) -> Result<Self> {
    let inner_size = u64::from(width)
      .checked_mul(u64::from(height))
      .and_then(|pixels| pixels.checked_mul(4))
      .and_then(|bytes| usize::try_from(bytes).ok())
      .ok_or_else(|| Error::new(Status::InvalidArg, "image dimensions are too large"))?;
    let external_size = i64::try_from(inner_size)
      .map_err(|_| Error::new(Status::InvalidArg, "image dimensions are too large"))?;

    let mut inner = Vec::new();
    inner.try_reserve_exact(inner_size).map_err(|err| {
      Error::new(
        Status::GenericFailure,
        format!("failed to allocate image buffer: {err}"),
      )
    })?;
    inner.resize(inner_size, 0);
    env.adjust_external_memory(external_size)?;
    Ok(Self {
      width,
      height,
      inner,
    })
  }
}

impl ObjectFinalize for CustomFinalize {
  fn finalize(self, mut env: Env) -> Result<()> {
    let external_size = i64::try_from(self.inner.len())
      .map_err(|_| Error::new(Status::InvalidArg, "image buffer is too large"))?;
    env.adjust_external_memory(-external_size)?;
    Ok(())
  }
}
```

Primeiro, vocĂŞ pode definir o atributo `custom_finalize` no macro `#[napi]`, e o NAPI-RS nĂŁo gerarĂĄ o `ObjectFinalize` padrĂŁo para a estrutura Rust.

EntĂŁo, vocĂŞ pode implementar o `ObjectFinalize` vocĂŞ mesmo para a Rust struct.

Neste caso, a estrutura `CustomFinalize` aumenta a memĂłria externa no **constructor** e a diminui em `fn finalize`.

## `instance of`

HĂĄ um `fn instance_of` em todas as classes `#[napi]`:

```rust {9} filename="lib.rs"

use napi_derive::napi;

#[napi(constructor)]
pub struct NativeClass {}

#[napi]
pub fn is_native_class_instance(env: &Env, value: Unknown) -> Result<bool> {
  NativeClass::instance_of(env, &value)
}
```

```js filename="main.mjs"

const nc = new NativeClass()
console.log(isNativeClassInstance(nc)) // true
console.log(isNativeClassInstance(1)) // false
```
