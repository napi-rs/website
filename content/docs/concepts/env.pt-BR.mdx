Na maioria dos casos, o Node-API fica encapsulado em várias abstrações e
estruturas de alto nível do **NAPI-RS**. No entanto, em alguns cenários, você
ainda precisa acessar o Node-API subjacente.

A struct `Env` fornece acesso ao ambiente do Node-API e permite criar valores
JavaScript, lidar com erros, gerenciar memória e interagir com o runtime
JavaScript.

## Criação de strings e symbols

### `create_string`

Cria uma string JavaScript a partir de um tipo Rust que pode ser convertido em
`&str`.

```rust
pub fn create_string<S: AsRef<str>>(&self, s: S) -> Result<JsString<'_>>
```

**Exemplo:**
```rust
let js_string = env.create_string("Hello, World!")?;
```

### `create_string_from_std`

Cria uma string JavaScript a partir de uma `String` do Rust.

```rust
pub fn create_string_from_std<'env>(&self, s: String) -> Result<JsString<'env>>
```

### `create_string_from_c_char`

Cria uma string JavaScript a partir de um ponteiro para string em estilo C.
Isso é usado em cenários de C FFI.

<Callout type="info">
  Você pode passar `NAPI_AUTO_LENGTH` como parâmetro `len` se a string C for
  terminada em nulo.
</Callout>

```rust
pub unsafe fn create_string_from_c_char<'env>(
  &self,
  data_ptr: *const c_char,
  len: isize,
) -> Result<JsString<'env>>
```

### `create_string_utf16`

Cria uma string JavaScript a partir de dados codificados em UTF-16.

```rust
pub fn create_string_utf16<C: AsRef<[u16]>>(&self, chars: C) -> Result<JsString<'_>>
```

```rust
pub fn create_string_latin1<C: AsRef<[u8]>>(&self, chars: C) -> Result<JsString<'_>>
```

## Criação de funções e classes

### `create_function`

Cria uma função JavaScript a partir de um callback nativo.

```rust
pub fn create_function<Args: JsValuesTupleIntoVec, Return>(
  &self,
  name: &str,
  callback: Callback,
) -> Result<Function<'_, Args, Return>>
```

**Exemplo:**

<Callout type="info">
  Você pode acessar o Callback **`C`** adicionando o sufixo `_c_callback` ao
  nome da função. No exemplo abaixo, `custom_function_c_callback` é o callback
  `C` de `custom_function`.
</Callout>

```rust {6,10} filename="lib.rs"
use napi::bindgen_prelude::*;
use napi_derive::napi;

#[napi]
pub fn create_function(env: &Env) -> Result<Function<u32, u32>> {
  env.create_function("customFunction", custom_function_c_callback)
}

#[napi(no_export)]
fn custom_function(input: u32) -> u32 {
  input * 2
}
```

<Callout type="info">
  O atributo `no_export` é usado para impedir que a função seja exportada para
  o lado JavaScript.
</Callout>

`custom_function` não é exportada, então ela não fica visível no lado
JavaScript. Mas o callback `C` é usado para criar `Function` em
`fn create_function`. Você pode usá-lo assim:

```ts filename="index.ts"
import { createFunction } from './index.js';

const customFunction = createFunction();
console.log(customFunction(2)); // 4
```

### `create_function_from_closure`

<Callout type="info">
  Requer a feature `napi5`.
</Callout>

Cria uma função JavaScript a partir de um closure Rust.

```rust
pub fn create_function_from_closure<Args: JsValuesTupleIntoVec, Return, F>(
  &self,
  name: &str,
  callback: F,
) -> Result<Function<'_, Args, Return>>
where
  Return: ToNapiValue,
  F: 'static + Fn(FunctionCallContext) -> Result<Return>,
```

**Exemplo:**

```rust {6,9} filename="lib.rs"
use napi::bindgen_prelude::*;
use napi_derive::napi;

#[napi]
pub fn create_function(env: &Env) -> Result<Function<u32, u32>> {
  let var_moved_into_closure = 42; // esta variável é movida para dentro do closure
  env.create_function_from_closure("rustClosure", move |ctx| {
    // obtém o primeiro argumento vindo do lado JavaScript
    let result = var_moved_into_closure + ctx.get::<u32>(0)?;
    Ok(result)
  })
}
```

```ts filename="index.ts"

console.log(rustClosure(2)); // 44
```

## Suporte a async e futures

### `spawn`


### `spawn_future`

<Callout type="info">

</Callout>

```rust
pub fn spawn_future<
  T: 'static + Send + ToNapiValue,
  F: 'static + Send + Future<Output = Result<T>>,

```

### `spawn_future_with_callback`

<Callout type="info">
  Requer as features `tokio_rt` e `napi4`.
</Callout>

```rust
pub fn spawn_future_with_callback<
  T: 'static + Send,
  V: ToNapiValue,
  F: 'static + Send + Future<Output = Result<T>>,
  R: 'static + FnOnce(Env, T) -> Result<V>,
```
