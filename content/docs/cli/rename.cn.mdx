# 重命名

import { Green, Rust } from '../../../components/chalk'

重命名 **NAPI-RS** 项目。

## 用法

```sh
# CLI
napi rename [--options]
```

```typescript
// Programmatically
import { NapiCli } from '@napi-rs/cli'

new NapiCli().rename({
  // options
})
```

## 选项

| 选项 | CLI 选项 | 类型 | 必填 | 默认值 | 描述 |
| --- | --- | --- | --- | --- | --- |
| | --help,-h | | | | 获取帮助 |
| cwd | --cwd | string | false | process.cwd() | napi 命令执行的工作目录，其他所有路径选项都相对于该路径 |
| configPath | --config-path,-c | string | false | | <Green>napi</Green> 配置 JSON 文件的路径 |
| packageJsonPath | --package-json-path | string | false | <Green>package.json</Green> | <Green>package.json</Green> 的路径 |
| npmDir | --npm-dir | string | false | <Green>npm</Green> | 存放 npm 包的目录路径 |
| name | --name,-n | string | false | | 项目的新名称 |
| binaryName | --binary-name,-b | string | false | | `*.node` 文件的新二进制名称 |
| packageName | --package-name | string | false | | 项目的新包名 |
| manifestPath | --manifest-path | string | false | <Rust>Cargo.toml</Rust> | <Rust>Cargo.toml</Rust> 的路径 |
| repository | --repository | string | false | | 项目的新仓库地址 |
| description | --description | string | false | | 项目的新描述 |
